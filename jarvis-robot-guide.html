// MODULO 01

🎙️ VOICE INPUT

Il microfono cattura la tua voce. Un sistema di Wake Word (es. "Hey Jarvis") attiva l'ascolto. Il parlato viene convertito in testo tramite Whisper o Google STT.

// MODULO 02

🧠 AI BRAIN

Il testo viene inviato all'API di Claude o GPT. La risposta generata viene convertita in audio (TTS) e riprodotta dallo speaker del robot.

// MODULO 03

🚗 MOVIMENTO

Il Pi gestisce i motori DC tramite un driver L298N connesso ai pin GPIO. L'AI può comandare movimenti dicendo "avanza", "gira a destra" etc.

// MODULO 04

⚡ ALIMENTAZIONE

Una batteria LiPo o un powerbank da 10.000 mAh alimenta il Pi. I motori hanno una batteria separata (6-12V) per evitare interferenze.

💡 APPROCCIO CONSIGLIATO:

Il Raspberry Pi 3 è sufficientemente potente per gestire la logica, ma la maggior parte dell'AI (LLM) verrà eseguita nel cloud tramite API (Claude, OpenAI). Per l'elaborazione locale puoi usare Whisper (tiny/base) per il riconoscimento vocale offline, mentre per il TTS pyttsx3 o Coqui-TTS funzionano senza internet.

💻 STACK SOFTWARE
// OS

Raspberry Pi OS (64-bit)

Usa la versione Lite (headless) per risparmiare RAM. Configura SSH e WiFi via raspi-config. Python 3.11+ è preinstallato.

// STT — SPEECH TO TEXT

OpenAI Whisper (tiny)

Modello tiny: 39M parametri, gira su Pi 3 in 3-8 secondi. Alternativa online: Google Speech-to-Text API (più veloce, richiede internet).

// AI BRAIN

Claude API (Anthropic)

Usa claude-haiku-4-5 per risposte rapide e a basso costo. Ottimo per comandi in tempo reale. GPT-4o-mini è un'alternativa valida.

// TTS — TEXT TO SPEECH

pyttsx3 / Coqui TTS

pyttsx3 funziona offline, è leggero ma robotico. Coqui TTS produce voce più naturale. ElevenLabs API per qualità cinema.

// WAKE WORD

Porcupine (Picovoice)

Rilevamento wake word offline, ultra-leggero (gira su MCU). Free tier disponibile. Alternativa: Snowboy (deprecato ma funziona) o keyword detection manuale.

// MOTORI

RPi.GPIO / gpiozero

Librerie Python per controllare i pin GPIO. gpiozero ha una classe Robot già pronta. pigpio offre PWM hardware più preciso per velocità variabile.

// AUDIO

PyAudio + ALSA

PyAudio per catturare audio dal microfono. Configura ALSA per selezionare il device corretto. sounddevice è un'alternativa più moderna.

// ORCHESTRAZIONE

Python asyncio

Gestisci microfono, AI e motori in modo asincrono. Il robot deve ascoltare mentre si muove. Un event loop centrale coordina tutto.

🔩 SCHEMA DI ASSEMBLAGGIO
01

COLLEGA I MOTORI AL DRIVER L298N

Motore sinistro → OUT1 e OUT2. Motore destro → OUT3 e OUT4. La batteria motori (6V) va su +12V e GND del L298N. Il jumper da 5V deve essere presente se la batteria è ≤ 12V.

02

COLLEGA L298N AI GPIO DEL RASPBERRY

IN1 → GPIO 17 (pin 11) | IN2 → GPIO 27 (pin 13) | IN3 → GPIO 22 (pin 15) | IN4 → GPIO 23 (pin 16) | ENA → GPIO 18 (PWM) | ENB → GPIO 25 (PWM). GND comune tra Pi e L298N!

GPIO Pi 3 Pinout: PIN 11 = GPIO17 → L298N IN1 PIN 13 = GPIO27 → L298N IN2 PIN 15 = GPIO22 → L298N IN3 PIN 16 = GPIO23 → L298N IN4 PIN 12 = GPIO18 → L298N ENA (PWM) PIN 22 = GPIO25 → L298N ENB
03

COLLEGA MICROFONO E SPEAKER

Il microfono USB si collega direttamente a una porta USB del Pi. Lo speaker con jack 3.5mm va nell'audio jack del Pi. Testa subito con arecord -l e aplay -l.

04

ALIMENTAZIONE

Powerbank USB → Micro-USB del Pi. Batteria motori separata → L298N. Mai collegare la batteria motori al Pi direttamente. Aggiungi un interruttore fisico per sicurezza.

05

ASSEMBLA IL TELAIO

Monta i motori al telaio, aggiungi le ruote. Posiziona il Pi sopra con distanziali M3. Il L298N va sotto o lateralmente. Lascia i cavi in ordine — userai breadboard e jumper wire per i prototipi.

🔌

GND COMUNE: È fondamentale che il GND del Raspberry Pi e il GND del driver L298N siano collegati insieme. Senza questo collegamento, i segnali GPIO non funzioneranno correttamente.

robot_motion.py — Controllo Motori

# robot_motion.py import RPi.GPIO as GPIO import time # Pin GPIO IN1, IN2, IN3, IN4 = 17, 27, 22, 23 ENA, ENB = 18, 25 GPIO.setmode(GPIO.BCM) GPIO.setup([IN1,IN2,IN3,IN4,ENA,ENB], GPIO.OUT) pwmA = GPIO.PWM(ENA, 100) pwmB = GPIO.PWM(ENB, 100) pwmA.start(75) # velocità 75% pwmB.start(75) def forward(): GPIO.output([IN1,IN3], GPIO.HIGH) GPIO.output([IN2,IN4], GPIO.LOW) def backward(): GPIO.output([IN2,IN4], GPIO.HIGH) GPIO.output([IN1,IN3], GPIO.LOW) def turn_left(): GPIO.output(IN1, GPIO.LOW) GPIO.output(IN2, GPIO.HIGH) GPIO.output(IN3, GPIO.HIGH) GPIO.output(IN4, GPIO.LOW) def turn_right(): GPIO.output(IN1, GPIO.HIGH) GPIO.output(IN2, GPIO.LOW) GPIO.output(IN3, GPIO.LOW) GPIO.output(IN4, GPIO.HIGH) def stop(): GPIO.output([IN1,IN2,IN3,IN4], GPIO.LOW)

jarvis_brain.py — AI Core

# jarvis_brain.py import anthropic import pyttsx3 import speech_recognition as sr client = anthropic.Anthropic(api_key="LA_TUA_CHIAVE") tts = pyttsx3.init() tts.setProperty('rate', 160) SYSTEM_PROMPT = """Sei JARVIS, l'assistente AI di un robot. Rispondi in italiano, in modo conciso. Quando vuoi che il robot si muova, aggiungi tag come: [AVANZA], [INDIETRO], [GIRA_SX], [GIRA_DX], [STOP]""" def ask_jarvis(user_input: str) -> str: response = client.messages.create( model="claude-haiku-4-5-20251001", max_tokens=300, system=SYSTEM_PROMPT, messages=[{"role":"user","content":user_input}] ) return response.content[0].text def speak(text: str): # Rimuovi i tag movimento prima di parlare clean = text.replace("[AVANZA]","").replace("[STOP]","") tts.say(clean) tts.runAndWait() def listen() -> str: r = sr.Recognizer() with sr.Microphone() as source: print("In ascolto...") audio = r.listen(source, timeout=5) return r.recognize_google(audio, language="it-IT")

main.py — Loop Principale

# main.py — Il cuore di JARVIS import re, time from robot_motion import forward, stop, turn_left, turn_right from jarvis_brain import ask_jarvis, speak, listen COMMANDS = { "AVANZA": (forward, 1.5), "INDIETRO": (backward, 1.5), "GIRA_SX": (turn_left, 0.8), "GIRA_DX": (turn_right, 0.8), "STOP": (stop, 0), } def execute_commands(response: str): tags = re.findall(r'\[(\w+)\]', response) for tag in tags: if tag in COMMANDS: action, duration = COMMANDS[tag] action() if duration > 0: time.sleep(duration) stop() def main(): speak("Sistema JARVIS attivato. Come posso aiutarti?") while True: try: user_input = listen() print(f"Tu: {user_input}") response = ask_jarvis(user_input) print(f"JARVIS: {response}") execute_commands(response) speak(response) except Exception as e: print(f"Errore: {e}") if __name__ == "__main__": main()
